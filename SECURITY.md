# Security Policy

Bitte Sicherheitslücken nicht als öffentliches Issue mit Exploitdetails melden.
Nutze stattdessen die private Funktion **Security → Report a vulnerability**
dieses GitHub-Repositories. Füge nur die zur Reproduktion nötigen Daten bei und
keine realen Kundendatensätze, Passwörter oder Tokens.

Hilfreich sind betroffene Version/Release-ID, Auswirkung, reproduzierbare
Schritte, erwartetes Verhalten und ein minimaler Nachweis. Der Eingang wird
triagiert; eine belastbare Frist oder Vergütung wird erst zugesagt, wenn Umfang
und Betreiberzuständigkeit geklärt sind.

Aktiv unterstützt wird ausschließlich der aktuell produktiv veröffentlichte
Release. Frühere Tags dienen der Nachvollziehbarkeit und erhalten grundsätzlich
keine separaten Fehlerbehebungen; eine Korrektur erscheint als neuer Release.

Das interne Verfahren steht in
[`docs/operations/incident-runbook.md`](docs/operations/incident-runbook.md).
